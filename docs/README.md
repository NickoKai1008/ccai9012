# Documentation Site Builder

This directory provides a lightweight static documentation site generation workflow:

- Put Markdown source files in `docs/md/`
- Output generated HTML pages to `docs/` (repository `docs/` is the site root)
- Homepage is generated to `docs/index.html` (GitHub Pages expects this)
- Sidebar navigation / page mapping is centrally configured in `docs/pages.json` (supports multi-level `children`, suitable for multi-level directories like `starter_kits/`)
- API docs do not go through Markdown: `docs/generate_api_doc.py` generates them directly into `docs/api/` (then `md_to_html.py` injects the site style and ensures navigation consistency)

## File structure and execution logic

- `docs/md/`: Markdown source files for site content (e.g., `docs/md/datasets.md`, `docs/md/starter_kits/index.md`)
- `docs/`: Final HTML output (site root)
  - Example: `docs/timetable.html`, `docs/starter_kits/index.html`
- `docs/index.html`: Homepage (generated from `docs/md/index.md`)
- `docs/api/`: API documentation output directory (generated by pdoc, not produced by Markdown conversion)
  - Example: `docs/api/ccai9012/index.html`
- `docs/pages.json`:
  - Defines the navigation hierarchy (`pages` + `children`)
  - Defines the Markdown → HTML mapping (`source` / `output`)
  - Can configure homepage cards (`home_card`), etc.

## Usage (recommended order)

### Step 1: Generate the API HTML docs for `ccai9012`

You can run this from either the repository root or the `docs/` directory (recommended to run under `docs/` since paths are more intuitive):

```bash
cd docs
python generate_api_doc.py
```

This step calls `pdoc` (or `pdoc3`) to generate HTML into:

- `docs/api/ccai9012/...`

### Step 2: Convert the Markdown under `docs/md/` into site HTML

```bash
cd docs
python md_to_html.py
```

This step will:

- Read `docs/pages.json` and convert each declared `source` (relative to `docs/md/`) into HTML
- Write all Markdown pages under `docs/` (the site root), including nested paths such as `docs/starter_kits/...`
- Generate a unified sidebar navigation and stylesheet references
- Process/unify some site-level style integration for pages under the API docs output directory (the API itself is still generated in Step 1)

## Adding new pages (supports multi-level directories, e.g., starter-kits)

When adding a new page, you need to do two things:

1) Add the corresponding Markdown file under `docs/md/`
- For example: `docs/md/weekly_syllabus.md`
- Or a multi-level structure: `docs/md/starter_kits/extra/topic_a.md`

2) Add a configuration node in the `pages` tree of `docs/pages.json`
- Key fields:
  - `key`: unique across the site (used for highlighting and references)
  - `label`: display name in the sidebar
  - `title`: HTML `<title>` and the page title
  - `source`: Markdown path relative to `docs/md/`
  - `output`: HTML output path (relative to `docs/` site root)
  - `children`: list of child pages (can be nested indefinitely; compatible with multi-level structures)
  - `home_card`: whether to show on the homepage card area (optional)

Notes:
- If you only want to add a navigation item that points to an “external/already generated page” (e.g., the API home), you can configure only `href` without `source`.
  - There is already an example in the repo: `key: "api"` uses `href: "api/ccai9012/index.html"`.
